## Run Locally

**Prerequisites:**  Node.js


1. Install dependencies:
   `npm install`
2. Set the `GEMINI_API_KEY` in [.env.local](.env.local) to your Gemini API key
3. Run the app:
   `npm run dev`
#Курсовий проєкт (Професійний сертифікат "Google AI")
Це інструмент для логістичної аналітики. Усі складні розрахунки відбуваються безпосередньо на комп'ютері користувача, тому жодні корпоративні дані (ціни чи маршрути) не передаються в інтернет.
У менеджера завжди є вибір:
Відправити вантаж своїми машинами (це надійно, але часто дорожче).
Залучити стороннього перевізника (це дешевше, але є ризик, що він запізниться або не приїде).
Це програма, яка допомагає швидко і максимально вигідно розподіляти вантажі між власними фурами та найманими перевізниками. Працівнику достатньо просто надати звичайну табличку зі своїми даними, і програма сама розрахує ідеальну пропорцію: кому і скільки рейсів віддати, щоб заощадити найбільше грошей і доставити все вчасно.
Технології: React, TypeScript, Tailwind CSS .
Він виконує такі функції:
Збір даних: Приймає скопійовані дані у вигляді таблиці (через textarea), завантаження CSV або ручне введення.
Парсинг: Очікує 6 ключових параметрів: назва маршруту, обсяг (прогноз рейсів), вартість власного авто, вартість найманого авто, SLA власного авто та SLA найманого.
Оптимізація та Симуляція: Перераховує квоти між власним і найманим транспортом, дозволяє симулювати зміни цін на пальне або тарифів ринку за допомогою повзунків.
Звітність: Генерує дешборди, таблиці з результатами, інсайти, ризики та готовий Markdown-звіт для копіювання.

##Розумний розподіл рейсів (Мозок програми)
Програма автоматично аналізує всі варіанти, як досвідчений директор з логістики:
Вона порівнює ціни та показники надійності (SLA).
Захищає від ризиків: не віддає всі рейси сторонньому водію, якщо він дешевий, але часто спізнюється. Програма завжди залишає частину рейсів для власних надійних авто як "страховку".
Шукає максимальну вигоду: там, де надійний партнер дає велику знижку, програма віддає йому максимум рейсів, збільшуючи прибуток компанії.
Ядро прийняття рішень (optimizeRouteDistribution)  реалізує алгоритм квотування (Decision Matrix) для кожного маршруту окремо. Вона працює на основі пріоритетів та жорстко запрограмованих правил (Rule-based engine):
Правило 1 (Ідеальний сценарій - rule_1_owned_better): Якщо власний транспорт дешевший і має кращий/рівний SLA — 100% обсягу віддається власному парку.
Правило 2 (Мінімізація ризиків - rule_2_hired_low_sla): Якщо найманий перевізник дешевший, але ненадійний (SLA < 90%), система блокує передачу йому всього обсягу. Віддається лише 30%, а 70% бронюється за власним авто для страховки.
Правило 3 (Максимізація маржі - rule_3_hired_super_saving): Якщо перевізник дає економію більше ніж 10% і при цьому надійний (SLA ≥ 95%), йому віддається 80% квот. Це генерує чистий прибуток.
Правило 4 (Спірні випадки): При однаковій ціні 100% віддається тому, у кого вищий SLA.
Резервні правила (Fallbacks): Алгоритм містить складну логіку для "сірих зон" (коли економія незначна, або показники якості змішані), забезпечуючи розподіл 60/40 або 80/20 на користь безпеки чи економії.
Математичне балансування: Наприкінці функція перетворює відсотки у фізичні рейси (Math.round), гарантуючи, що сума рейсів збігається з початковим прогнозом, навіть якщо обсяг дуже малий (наприклад, всього 1-3 рейси).

##Модуль агрегації (calculateOptimizationSummary)
Збирає результати по всіх маршрутах в єдиний дашборд.
Рахує загальну кількість рейсів (всього, власних, найманих).
Головна фінансова метрика (netSavings): Розраховує різницю між baselineCost (ніби ми все повезли своїми дорогими машинами) та optimizedCost (нашим розумним міксом). Це і є ті гроші, які додаток "зекономив" бізнесу.
Рахує середньозважені показники SLA (Weighted Average), щоб показати загальний показник "здоров'я" логістичної мережі.
3. Симулятор "Що, якщо?"
Логіст може потягнути за повзунки на екрані, уявивши ситуацію: "А що буде, якщо завтра пальне подорожчає на 10%?" або "Що, як перевізники піднімуть тарифи?". Програма миттєво перерахує весь план дій без перезавантаження сторінки.

##Готові текстові звіти
Менеджеру не треба писати звіти самостійно. Програма сама формує зрозумілі висновки текстом. Наприклад: "Увага, на маршруті Київ-Львів є ризик зриву поставок через поганого перевізника". Цей звіт можна скопіювати в один клік і відправити керівництву.

##Бізнес-цінність
Цей додаток — генератор реальної економії для компанії:
Економія без втрати якості: Компанія суттєво зменшує логістичні витрати, але при цьому гарантує, що всі товари доїдуть до клієнтів вчасно.
Мінімізація ризиків: Розумне бронювання рейсів за власним автопарком гарантує, що компанія не платитиме штрафи мережам магазинів через безвідповідальних підрядників.
Більше прибутку: Знаходячи ідеальні комбінації ціни та якості на сотнях маршрутів, програма дозволяє збільшувати чистий прибуток компанії на кожному відправленому автомобілі.
Від годин до секунд: Працівники можуть приділити час більш важливим задачам, а не рутинним таблицям.
Безпека компанії: Оскільки додаток виконує всі математичні розрахунки прямо в браузері працівника, комерційні таємниці (ціни, тарифи, обсяги перевезень) ніколи не залишають межі комп'ютера компанії. Їх неможливо перехопити чи вкрасти.
